Aurreum's Continuous Log Real-time Protection (CLRP) technology - achieving sub-second RPO is not a dream

Thu, May 25, 2023

How to reduce the RPO to sub-second level for backup and recovery of Oracle databases?

How to achieve real-time data replication while minimizing resource consumption?

Aurreum’s Continuous Log Real-time Protection (CLRP) technology effortlessly addresses these challenges. Let’s explore the advantages and principles behind this technology:

It is well-known that Oracle database backups are typically performed using the RMAN backup tool. However, RMAN can only backup data files and archived logs. Consequently, in the event of an issue with the production host, the restoration can only be performed to the state of the most recent log archiving. Data loss in such cases is typically in the range of hours or even half a day, resulting in an RPO of several hours or more. This situation fails to meet the data protection requirements of customers.

To enhance the real-time nature of data protection, Aurreum has independently developed the Continuous Log Real-time Protection (CLRP) technology. CLRP is based on block-level backup, continuously monitoring changes to redo logs, archived logs, and other files through filesystem message monitoring, and performing real-time continuous backups of log files based on these changes.

The CLRP technology, built upon periodic full database backups, performs real-time replication of redo logs and archived logs of the production database, achieving backup protection of changed data within sub-second intervals. This advancement elevates database protection technology to new heights, reducing the previously several-hour or more RPO to sub-second levels, with an average of 0.2 seconds under normal load conditions.

When utilizing the CLRP technology, there is no need to install any software or hardware drivers, and it does not impact the operation of the business system. On the other hand, some competing solutions claiming to support continuous log backup require the deployment of drivers at the system level, which can adversely affect system stability if any monitoring issues arise.

Furthermore, most solutions on the market adopt volume-level backups, which cannot selectively backup only the changed data blocks in Oracle or MySQL applications, resulting in a significantly larger backup data block size compared to the actual changed data blocks in Oracle. This significantly increases the resource consumption on the client side, impacting the operation of business services, particularly in busy databases.

By employing the CLRP technology, users can restore the database to any desired time point or SCN (System Change Number). Due to the extremely small size of the real-time replicated changed data blocks, the individual space consumption is almost negligible, enabling recovery to any desired time point or SCN.

In summary, utilizing the Continuous Log Real-time Protection (CLRP) technology brings notable advantages:

  • Reducing the traditional backup solution’s RPO from several hours or more to sub-second levels, with an average of 0.2 seconds under normal load conditions.
  • Compared to Continuous Data Protection (CDP) solutions, the CLRP technology’s real-time replication of data blocks is significantly smaller, resulting in lower resource consumption compared to other CDP solutions.
  • Deploying CLRP requires no installation of software or hardware drivers on the production host, with minimal impact on the business system running on the production host.
  • It enables database recovery to any desired time point or SCN, effectively addressing any extreme software or hardware failures in the production system.
